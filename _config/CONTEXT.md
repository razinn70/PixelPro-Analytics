# Config Layer

Stable identity, design, and security configuration for all stages.

## Files

| File | Purpose |
|------|---------|
| brand-identity.md | PixelPro brand positioning, voice, service offering |
| design-system.md | Colors, typography, spacing, component tokens |
| client-verticals.md | Restaurant / ecommerce / service starter templates |
| security-policy.md | Auth, RLS, CSP, data privacy rules |

## Usage

All stages READ from this directory. No stage writes to `_config/`.
Update these files when brand, design, or security policy changes globally.
